# Simple Job Scheduler

**English:** [readme.en.md](readme.en.md)

HTTP-планировщик задач на Rust: расписание (интервал / cron / один раз), пайплайн fetch → JS-преобразование → send, SQLite, веб-интерфейс на Vue + Tailwind (локальная статика, без сборки и без CDN).

## Возможности

- REST API и веб-UI: обзор, список задач, форма создания/редактирования, журнал выполнения.
- Типы расписания: интервал (`5m`, `2h`, `1d`), cron (5 полей, UTC), однократный запуск.
- Пайплайн задачи: HTTP fetch → опциональный JS (boa) → HTTP send.
- Повторы при ошибках (настраиваемые лимит и интервал).
- Планировщик с тик-циклом, ограничение параллельных запусков, ручной запуск.
- SQLite: задачи и история выполнения; автоочистка старых записей журнала.
- Серверные логи в stdout и в файлы с ротацией.
- Локализация UI и служебных сообщений (`ru` / `en`).
- Веб-UI на Vue 3 и Tailwind: все ассеты лежат в `web/`, Node.js не нужен.
- Валидация полей задачи на сервере (проверяются только включённые шаги).

## Требования

- Rust (stable) и Cargo

## Запуск

```bash
cp .env.example .env
cargo run
```

Сервер слушает `AJS_HOST:AJS_PORT`. В `.env.example` по умолчанию `127.0.0.1:3000`; без `.env` в коде используется порт `6378`.

Откройте в браузере: [http://127.0.0.1:3000](http://127.0.0.1:3000) (или ваш порт из `.env`).

## Сборка портативной версии

```bash
cargo build --release
```

### Остановка

**Ctrl+C** (или закрытие консоли с сигналом прерывания) запускает корректное завершение:

1. HTTP-сервер перестаёт принимать новые соединения и дожидается активных запросов (`graceful shutdown`).
2. Останавливается тик-цикл планировщика; новые запуски по расписанию и кнопке «Запустить» не начинаются.
3. Активные задачи **прерываются** (отмена HTTP и фоновых `tokio`-задач), ожидание снятия блокировок до 5 с, затем закрытие пула SQLite (таймаут 5 с).

В логах появятся сообщения о получении сигнала и успешном завершении.

## Переменные окружения

Префикс **`AJS_`** (см. `.env.example`):

| Переменная                           | Назначение                                                                                                                        | По умолчанию     |
| ------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------- | ---------------- |
| `AJS_HOST`                           | Адрес bind HTTP-сервера                                                                                                           | `127.0.0.1`      |
| `AJS_PORT`                           | Порт HTTP-сервера                                                                                                                 | `6378`           |
| `AJS_DB_PATH`                        | Файл SQLite                                                                                                                       | `./scheduler.db` |
| `AJS_LOG_LEVEL`                      | Уровень логов (`tracing`)                                                                                                         | `info`           |
| `AJS_DEFAULT_LANGUAGE`               | Язык сервера и UI по умолчанию: `ru` или `en`                                                                                     | `en`             |
| `AJS_MAX_CONCURRENT_JOBS`            | Макс. параллельных выполнений                                                                                                     | `10`             |
| `AJS_HTTP_TIMEOUT_SECONDS`           | Таймаут HTTP fetch/send (с)                                                                                                       | `60`             |
| `AJS_JOB_TICK_INTERVAL_MS`           | Интервал тика планировщика (мс)                                                                                                   | `1000`           |
| `AJS_ENABLE_JS_TRANSFORM`            | Разрешить JS-преобразование (`true`/`false`)                                                                                      | `true`           |
| `AJS_RETENTION_DAYS`                 | Хранение записей журнала выполнения (дни)                                                                                         | `30`             |
| `AJS_LOG_RESPONSE_PREVIEW_MAX_BYTES` | Макс. размер поля `response_preview` в журнале SQLite (байты UTF-8)                                                               | `500`            |
| `AJS_LOG_DIR`                        | Каталог файловых логов (относительно cwd или абсолютный путь)                                                                     | `./logs`         |
| `AJS_RUN_OVERDUE_ON_STARTUP`         | Выполнять просроченные задачи сразу после старта (`true`) или пересчитать `next_run_at` от текущего момента без запуска (`false`) | `true`           |
| `AJS_DISABLE_ALL_JOBS_ON_STARTUP`    | При старте отключить все задачи в БД (`enabled = 0`); включение — вручную в интерфейсе                                            | `false`          |

### Поведение при старте

Порядок при запуске: миграции → инициализация пустых `next_run_at` → **политика старта** → тик планировщика.

- **`AJS_DISABLE_ALL_JOBS_ON_STARTUP=true`** — все задачи сохраняются, но выключены; планировщик их не трогает, пока вы не включите нужные в UI.
- **`AJS_RUN_OVERDUE_ON_STARTUP=false`** — для **включённых** задач с `next_run_at` в прошлом запуск **не** выполняется; вместо этого `next_run_at` пересчитывается от «сейчас» (интервал/cron — следующий слот, one-time в прошлом — `next_run_at` сбрасывается).
- **`AJS_RUN_OVERDUE_ON_STARTUP=true`** (по умолчанию) — просроченные включённые задачи попадают в первый тик и выполняются (как раньше).

Типичная связка для ручного контроля: `AJS_DISABLE_ALL_JOBS_ON_STARTUP=true` и при необходимости `AJS_RUN_OVERDUE_ON_STARTUP=false`.

## Логирование

- **Серверные логи** (`tracing`):
  - stdout (с ANSI)
  - файл `scheduler.log` в `AJS_LOG_DIR`
- **Ротация файлов:**
  - размер одного файла: **2 МБ**
  - архивных файлов: до **10** (плюс активный)
  - схема имён: `scheduler.log`, `scheduler.1.log`, `scheduler.2.log`, …
- **Журнал выполнения задач** хранится в SQLite (`execution_logs`), не в файлах логов.
  - В таблице поле **`response_preview`** — укороченный фрагмент ответа для UI (лимит `AJS_LOG_RESPONSE_PREVIEW_MAX_BYTES`, по умолчанию 500 байт). При превышении в конец добавляется `…`, в БД пишется флаг `preview_truncated`.
  - Пайплайн (fetch → transform → send) работает с **полным** телом HTTP в памяти; в базу целиком оно **не** сохраняется.

## REST API

Базовый префикс: `/api`. Тело запросов — JSON.

| Метод    | Путь                      | Назначение                                                          |
| -------- | ------------------------- | ------------------------------------------------------------------- |
| `GET`    | `/api/dashboard`          | Статистика и последние запуски                                      |
| `GET`    | `/api/settings`           | Публичные настройки для UI (лимит превью журнала)                   |
| `GET`    | `/api/jobs`               | Список задач                                                        |
| `POST`   | `/api/jobs`               | Создать задачу                                                      |
| `PUT`    | `/api/jobs/{id}`          | Обновить задачу                                                     |
| `DELETE` | `/api/jobs/{id}`          | Удалить задачу                                                      |
| `POST`   | `/api/jobs/{id}/run`      | Запустить сейчас (фон)                                              |
| `POST`   | `/api/jobs/group-enabled` | Включить/выключить все задачи группы (`{ "job_group", "enabled" }`) |
| `GET`    | `/api/jobs/{id}/logs`     | История выполнения (до 100 записей)                                 |

Статика: `/` — веб-UI, `/i18n/{lang}.json` — строки интерфейса.

## Веб-интерфейс

Фронтенд — одностраничное приложение без этапа сборки: Axum отдаёт каталог `web/` как статику.

| Файл                 | Назначение                           |
| -------------------- | ------------------------------------ |
| `index.html`         | Разметка и Vue-приложение            |
| `vue.global.prod.js` | Vue 3 (production build)             |
| `tailwindcss.js`     | Tailwind CSS (runtime)               |
| `cron-formatter.js`  | Расшифровка cron и ближайшие запуски |

Скрипты подключаются локально из корня сайта (`/vue.global.prod.js` и т.д.). Для работы UI достаточно запущенного сервера — **интернет в браузере не требуется**. Переключение языка: `ru` / `en` через JSON в `i18n/`.

В разделе **Задачи**: необязательное поле **группа** (хранится в БД), фильтры по названию, группе и дате создания, сортировка; для выбранной группы — кнопки «Включить группу» / «Выключить группу».

### Пайплайн задачи

1. **Fetch** (если включён) — HTTP GET/POST, тело ответа → JSON для следующих шагов.
2. **Transform** (если включён) — JS в песочнице boa: переменная `input`, результат — `return …`.
3. **Send** (если включён) — HTTP POST/PUT; в шаблоне тела `{{payload}}` подставляется результат.

При ошибке на любом шаге — повторы (если включены), затем запись в журнал и пересчёт `next_run_at`.

### Валидация при сохранении

Проверяются только поля **включённых** секций:

- **Общее:** непустое имя.
- **Расписание:** интервал / cron / дата one-time (в будущем).
- **Fetch:** URL `http(s)://`, метод GET/POST, заголовки — JSON-объект.
- **Transform:** непустой скрипт.
- **Send:** URL `http(s)://`, метод POST/PUT, заголовки — JSON-объект.
- **Retry:** `max_retries ≥ 0`, `retry_interval_seconds ≥ 1`.

Ошибки возвращаются с кодом **400** и полем `error` (язык — `AJS_DEFAULT_LANGUAGE`).

## Примеры

### Создание задачи (интервал 5 минут, только fetch)

```bash
curl -s -X POST http://127.0.0.1:3000/api/jobs \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Ping API",
    "enabled": true,
    "schedule_type": "interval",
    "schedule_value": "5m",
    "fetch_enabled": true,
    "fetch_method": "GET",
    "fetch_url": "https://httpbin.org/get",
    "fetch_headers": "{}",
    "transform_enabled": false,
    "send_enabled": false,
    "retry_enabled": true,
    "max_retries": 2,
    "retry_interval_seconds": 30
  }'
```

### Ручной запуск

```bash
curl -s -X POST http://127.0.0.1:3000/api/jobs/{id}/run
```

### Cron (каждые 6 часов, UTC)

```json
"schedule_type": "cron",
"schedule_value": "0 */6 * * *"
```

## Ограничения

- JS-преобразование: нет Node.js API (сеть, файлы, `require`); каждый запуск — новый контекст boa.
- Cron и one-time расписания интерпретируются в **UTC**.
- Сравнение `next_run_at` в SQLite — лексикографическое по RFC3339; форматы дат должны быть согласованы.
- Одна строка cron — 5 полей (минута, час, день месяца, месяц, день недели), как в классическом Unix cron.
